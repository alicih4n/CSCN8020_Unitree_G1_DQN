# How to Run the MuJoCo Simulations on Mac M1

Your Mac M1 is perfectly capable of running MuJoCo natively. You do not need WSL or Ubuntu. Just open your VS Code Terminal, ensure your virtual environment is active (`source .venv/bin/activate`), and run the following commands:

> [!IMPORTANT]
> **macOS Cocoa Event Loop Requirement:**
> On macOS, any Python script that displays a MuJoCo GUI must be executed using **`mjpython`** instead of standard `python`. This is a macOS-specific constraint because the GUI event loop must run on the main thread. `mjpython` is a wrapper script automatically installed with the `mujoco` package.

### 1. Test the Viewer (Falling Box)
To ensure MuJoCo graphics work on your Mac:
```bash
mjpython src/test_mujoco_viewer.py
```
*This command runs a basic test of the passive MuJoCo viewer, loading a simple box falling onto a floor to verify that your screen/window renderer and windowing system work correctly on macOS.*

### 2. View the Original G1 Robot
To see the full floating-base robot (it might fall over):
```bash
mjpython src/inspect_g1_model.py external/unitree_mujoco/unitree_robots/g1/scene_29dof.xml
```
*This command opens the original G1 robot XML model, which is configured as a floating-base model. Without active standing control, the robot will naturally fall over under gravity when the simulation starts.*

### 3. View the Fixed-Base G1 Robot
To see the robot suspended in the air (ready for the elbow exercise):
```bash
mjpython src/inspect_g1_model.py assets/g1_fixed_base/scene_29dof_fixed_base.xml
```
*This command loads the fixed-base robot model generated by removing the floating base/free joint. The pelvis is pinned in the air, allowing you to safely control individual joints without the entire robot falling down.*

### 4. Run the Single-Joint Control (Elbow Movement)
To watch the PD controller move the left elbow to -0.8 radians:
```bash
mjpython src/control_single_joint.py --scene assets/g1_fixed_base/scene_29dof_fixed_base.xml --target -0.8 --duration 5
```
*This script uses a Proportional-Derivative (PD) controller to actuate the left elbow joint of the fixed-base G1 robot, driving it to a target angle of -0.8 radians over a duration of 5 seconds.*

### 5. Run the Interactive Gymnasium Demo (The Final Milestone)
This is the final demonstration where you can use your mouse to pan/zoom before starting the RL environment:
```bash
PYTHONPATH=src mjpython src/demo_g1_elbow_env.py
```
*(When the window opens, go back to the terminal and press Enter to start the movement).*

*This command runs the Gymnasium environment wrapper demo, integrating the Unitree G1 robot with standard Gym APIs. When launched, the simulation pauses to allow you to interactively position the camera. Once you press Enter in your terminal, the environment begins stepping and executing the joint movements.*
